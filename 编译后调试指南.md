## 编译后的操作流程

### 1. 检查编译产物

编译完成后，IPK文件会生成在以下位置：

```
output/luci-app-easystart_1.0-2_all.ipk
```

检查文件是否存在：

```bash
ls -la output/
```

### 2. 安装到路由器

使用提供的安装脚本进行安装：

```bash
# 在WSL Ubuntu中执行
cd /mnt/c/Users/haha/Documents/trae_projects/cudy
./install-to-router.sh
```

安装脚本会自动：
- 检查并卸载旧版本
- 上传IPK文件到路由器
- 安装新版本
- 重启LuCI服务
- 清理临时文件

### 3. 访问Web界面

安装完成后，访问路由器Web界面：

```
http://192.168.89.128/
```

进入 **网络 → 简易设置** 菜单查看插件是否正常显示。

## 调试步骤

### 1. 检查插件是否正确安装

登录路由器，执行以下命令：

```bash
# 检查插件是否已安装
opkg list-installed | grep luci-app-easystart

# 检查文件是否正确安装
ls -la /usr/lib/lua/luci/controller/
echo "---"
ls -la /usr/lib/lua/luci/model/cbi/easystart/
echo "---"
ls -la /usr/lib/lua/luci/view/easystart/
echo "---"
ls -la /etc/config/
echo "---"
ls -la /usr/bin/ | grep easystart
```

### 2. 检查LuCI错误日志

如果Web界面出现500错误或其他问题，查看LuCI日志：

```bash
# 查看LuCI错误日志
cat /var/log/luci.log

# 查看系统日志
logread
```

### 3. 检查配置文件

检查配置文件是否正确：

```bash
# 查看插件配置文件
cat /etc/config/easystart

# 检查网络配置
cat /etc/config/network

# 检查防火墙配置
cat /etc/config/firewall

# 检查DHCP配置
cat /etc/config/dhcp
```

### 4. 测试模式切换脚本

手动测试模式切换脚本：

```bash
# 查看脚本帮助
easystart-switch.sh --help

# 测试脚本执行
easystart-switch.sh --test

# 查看脚本执行权限
ls -la /usr/bin/easystart-switch.sh

# 确保脚本可执行
chmod +x /usr/bin/easystart-switch.sh
```

### 5. 网络服务状态

检查网络服务状态：

```bash
# 查看网络服务状态
/etc/init.d/network status

# 查看DHCP服务状态
/etc/init.d/dnsmasq status

# 查看防火墙状态
/etc/init.d/firewall status
```

### 6. 手动重启服务

如果服务异常，手动重启：

```bash
# 重启网络服务
/etc/init.d/network restart

# 重启DHCP服务
/etc/init.d/dnsmasq restart

# 重启防火墙
/etc/init.d/firewall restart

# 重启LuCI服务
/etc/init.d/uhttpd restart
```

## 常见问题及解决方案

### 1. Web界面500错误

**症状**：访问"简易设置"菜单时出现500内部服务器错误

**解决方案**：
- 检查CBI模型文件是否正确
- 检查LuCI日志中的具体错误信息
- 确保所有Lua文件语法正确

### 2. 模式切换失败

**症状**：点击应用后网络服务未正确重启

**解决方案**：
- 检查模式切换脚本是否可执行
- 查看脚本执行日志
- 手动执行脚本并观察输出

### 3. 网络连接问题

**症状**：切换模式后无法连接网络

**解决方案**：
- 检查网络配置文件
- 确保网关和DNS设置正确
- 重启网络服务

### 4. 插件未显示在菜单中

**症状**：安装后在Web界面中看不到"简易设置"菜单

**解决方案**：
- 重启LuCI服务
- 清除浏览器缓存
- 检查控制器文件是否正确安装

### 5. 权限问题

**症状**：脚本执行失败，提示权限不足

**解决方案**：
- 确保脚本具有执行权限：`chmod +x /usr/bin/easystart-switch.sh`
- 以root用户执行脚本

## 调试工具

### 1. LuCI调试模式

开启LuCI调试模式：

```bash
# 编辑uhttpd配置
vi /etc/config/uhttpd

# 在config uhttpd 'main'部分添加
option error_page '/luci-static/resources/error/500.html'

# 重启uhttpd
/etc/init.d/uhttpd restart
```

### 2. 网络诊断工具

```bash
# 检查网络连通性
ping 192.168.89.1

# 检查DNS解析
nslookup baidu.com

# 检查路由表
route -n

# 检查网络接口
ifconfig -a
```

### 3. 系统资源监控

```bash
# 查看系统负载
uptime

# 查看内存使用
free -h

# 查看磁盘使用
df -h

# 查看进程
ps aux | grep luci
```

## 回滚操作

如果遇到严重问题，需要回滚到之前的配置：

```bash
# 恢复网络配置
cp /etc/config/network.bak /etc/config/network

# 恢复防火墙配置
cp /etc/config/firewall.bak /etc/config/firewall

# 恢复DHCP配置
cp /etc/config/dhcp.bak /etc/config/dhcp

# 重启网络服务
/etc/init.d/network restart
```

## 总结

编译后调试是确保插件正常工作的重要步骤。通过以上调试方法，您可以快速定位和解决插件在运行过程中遇到的问题。

如果遇到无法解决的问题，请检查：
1. 编译过程是否有错误
2. 安装过程是否成功
3. 配置文件是否正确
4. 网络服务是否正常运行

通过系统性的调试和排查，您可以确保luci-app-easystart插件在OpenWrt路由器上稳定运行。
