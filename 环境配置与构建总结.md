## 构建流程

### GitHub Actions 构建配置

构建脚本位于 `.github/workflows/build-x86.yml`，主要步骤如下：

1. **环境准备**：安装必要的构建依赖
2. **添加交换空间**：创建 2GB 交换文件以避免内存不足
3. **克隆 OpenWrt 源码**：使用浅克隆方式克隆 OpenWrt 23.05 分支
4. **缓存配置**：缓存下载文件、构建目录和 feeds 目录以加快构建速度
5. **下载预编译 SDK**：使用预编译的 SDK 避免编译 toolchain 时的内存问题
6. **添加插件**：将 luci-app-easystart 插件复制到 OpenWrt 源码中
7. **更新 feeds**：更新并安装所有 feeds
8. **配置构建**：设置目标架构和插件选项
9. **编译插件**：使用单线程编译插件及其依赖
10. **上传构建产物**：将编译生成的 IPK 文件作为 artifact 上传
11. **创建发布**：自动创建 GitHub 发布版本

### 构建优化

1. **使用预编译 SDK**：避免编译 toolchain 时的内存不足问题
2. **缓存策略**：缓存下载文件、构建目录和 feeds 目录以加快构建速度
3. **单线程编译**：使用 -j1 参数避免内存溢出
4. **详细错误检查**：添加详细的错误检查和日志输出以便排查问题
5. **交换空间**：创建 2GB 交换文件以应对内存不足情况

## 插件功能

### 支持的模式

1. **传统路由模式**：标准路由功能，支持 PPPoE、DHCP 和静态 IP 配置
2. **旁路由模式**：作为网络中的辅助路由器，负责 DHCP 和 DNS 服务
3. **桥接模式**：将路由器变为纯 AP/交换机，关闭 DHCP 服务

### 操作流程

1. 在 LuCI 界面中选择 "简易设置" 菜单
2. 选择所需的工作模式
3. 根据模式类型填写相应的配置参数
4. 点击 "应用" 按钮执行模式切换
5. 系统会自动备份当前配置并应用新配置
6. 网络服务会自动重启以应用新配置

## 故障排查

### 常见问题

1. **内存不足**：使用预编译 SDK 并添加交换空间
2. **构建失败**：检查 SDK 下载是否成功，查看详细的构建日志
3. **模式切换失败**：检查脚本执行权限，查看脚本输出日志
4. **网络连接问题**：检查网络配置是否正确，重启网络服务

### 日志查看

- 构建日志：GitHub Actions 构建页面
- 插件日志：路由器系统日志或执行脚本时的输出

## 部署方式

1. **通过 GitHub Releases 下载**：从 GitHub 仓库的 Releases 页面下载编译好的 IPK 文件
2. **手动安装**：使用 `opkg install luci-app-easystart_1.0_all.ipk` 命令安装
3. **通过 OpenWrt 构建系统**：将插件添加到 OpenWrt 源码中一起编译
